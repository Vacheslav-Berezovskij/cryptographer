# MAX DH E2E Helper (Manifest V3)

Локальное браузерное расширение для одноразового E2E-обмена в MAX:
- ephemeral DH на каждое сообщение,
- `X25519` (если доступно) с fallback на `ECDH P-256`,
- `HKDF-SHA-256` -> `AES-256-GCM`,
- copy/paste протокол без внешних серверов и CDN.

> Важно: это **дополнение** к обычному чату. Метаданные (кто/когда/частота/размер) остаются видимыми сервису.

---

## 1) Что в проекте

- `manifest.json` — конфигурация MV3.
- `background.js` — service worker (bridge storage/runtime между popup и content script).
- `popup.html`, `popup.css`, `popup.js` — UI для ролей Отправитель/Получатель.
- `content-script.js` — интеграция с DOM MAX (кнопки импорта/вставки).
- `crypto.js` — ключи, ECDH/X25519, HKDF, base64 helpers.
- `cipher.js` — AES-GCM encrypt/decrypt, packet build/parse.
- `state-machine.js` — строгие шаги протокола и таймауты.
- `tests/browser-tests.html`, `tests/run-tests.js` — browser unit-тесты.
- `SECURITY_CHECKLIST_RU.md` — памятка безопасности.

## 2) Установка (Developer Mode)

1. Скачайте/клонируйте репозиторий локально.
2. Откройте страницу расширений в браузере Chromium:
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
3. Включите **Developer mode**.
4. Нажмите **Load unpacked** и выберите папку проекта.
5. Убедитесь, что расширение появилось в списке и popup открывается.

### Разрешения

- `storage` — только для вспомогательных payload/настроек UX.
- `clipboardWrite` — копирование подготовленных сообщений из UI.
- `host_permissions` на `max.ru`/`*.max.ru` — чтобы работал content script в веб-версии MAX.

## 3) Как использовать

### Роль «Отправить»

1. Откройте popup -> вкладка **Отправить**.
2. Нажмите **Новая сессия (params)**.
3. Отправьте выданный JSON в MAX.
4. Вставьте `ack_params` от собеседника -> **Следующий шаг**.
5. Отправьте `pub` в MAX.
6. Вставьте `pub` от собеседника.
7. Введите plaintext и нажмите **Следующий шаг**.
8. Отправьте final packet (base64 JSON) в MAX.
9. Нажмите **Сбросить сессию**.

### Роль «Читать»

1. Откройте popup -> вкладка **Читать**.
2. Вставьте `params` -> **Следующий шаг** (получите `ack_params`).
3. Отправьте `ack_params` в MAX.
4. Вставьте `pub` от отправителя -> **Следующий шаг** (получите `pub`).
5. Отправьте `pub` в MAX.
6. Вставьте final packet -> **Следующий шаг**.
7. Прочитайте расшифрованный текст.
8. Нажмите **Сбросить сессию**.

---

## 4) Content script в MAX

На странице MAX расширение:
- ищет служебные сообщения (`maxdh:` или protocol JSON),
- добавляет кнопку **«Импортировать в расширение»**,
- добавляет кнопку **«Сформировать сообщение»** у поля ввода.

Это упрощает перенос payload между чатом и popup.

## 5) Запуск тестов локально

### Browser unit tests

```bash
python3 -m http.server 8000
```

Откройте:

```text
http://localhost:8000/tests/browser-tests.html
```

Тесты покрывают:
- ключи и X25519/P-256 compatibility,
- `deriveSharedSecret` на двух сторонах,
- стабильность HKDF,
- AES-GCM encrypt/decrypt + tamper detection,
- полный прогон state machine.

---

## 6) Безопасность (обязательно)

- Никогда не отправляйте приватные значения/ключи в чат.
- При подозрении на MITM: остановить обмен -> сбросить сессию -> сверить вне канала -> начать заново.
- Для каждого сообщения использовать новую сессию.
- После завершения очищать сессию в popup.

Подробная памятка: `SECURITY_CHECKLIST_RU.md`.

---

## 7) Ограничения

- Это не скрывает метаданные чата.
- Надёжность зависит от корректности ручного copy/paste и валидации шагов.
- Поддержка `X25519` зависит от браузера; при недоступности используется `P-256`.

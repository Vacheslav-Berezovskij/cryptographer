# MAX DH E2E — чеклист безопасности и инструкции (RU)

## 1) Чеклист для пользователей (перед началом)

- [ ] Убедитесь, что используете официальную сборку расширения (из доверенного источника).
- [ ] Убедитесь, что у вас открыта правильная вкладка чата и правильный собеседник.
- [ ] Проверяйте, что отправляете только служебные **публичные** данные (`params`, `ack_params`, `pub`, `ciphertext`).
- [ ] Никогда не отправляйте в чат приватные значения, приватные ключи и промежуточные секреты.
- [ ] Для каждого нового сообщения запускайте новую одноразовую сессию.
- [ ] После завершения шага шифрования/расшифровки — сбросьте сессию в popup.

---

## 2) Как безопасно проходить обмен

1. **Обменивайтесь только тем, что просит интерфейс**: `params`, `ack_params`, `pub`, финальный `ciphertext`.
2. **Приватные значения не покидают расширение**: они не должны копироваться в буфер, чат, заметки и т.д.
3. **Сверяйте контекст**:
   - правильный собеседник,
   - правильный чат,
   - нет старых/повторных сообщений из прошлой сессии.
4. **Не переиспользуйте данные сессии** между разными сообщениями.

---

## 3) Что делать при подозрении на MITM

Признаки:
- внезапные ошибки валидации/расшифровки,
- неожиданные изменения параметров,
- несоответствие ожидаемого шага/состояния.

Действия:
1. **Немедленно остановите обмен** и не отправляйте новые payload.
2. Нажмите **«Сбросить сессию»** у обеих сторон.
3. Проведите сверку по независимому каналу (звонок/другой мессенджер):
   - кто инициатор,
   - какие шаги уже пройдены,
   - корректный ли собеседник.
4. Запустите полностью **новую** сессию с новыми ephemeral ключами.
5. Если ситуация повторяется — считать канал недоверенным и не передавать чувствительные данные.

---

## 4) Почему нельзя копировать приватные значения в общий чат

- Приватный ключ/секрет в чате = фактическая компрометация шифрования.
- Любой, кто увидит приватные данные (участники, админы, утечки, скриншоты), потенциально сможет расшифровать сообщения.
- Даже удалённые сообщения могут остаться в:
  - истории уведомлений,
  - резервных копиях,
  - локальном кеше,
  - системных логах.

**Правило:** в чат отправляются только публичные и транспортные данные протокола.

---

## 5) Как удалить следы сессии в расширении

Для пользователя:
1. В popup нажмите **«Сбросить сессию»** (вкладка «Отправить»/«Читать»).
2. Закройте popup (при закрытии память сессии очищается).
3. При необходимости перезагрузите вкладку MAX.
4. Очистите буфер обмена (вставьте случайный безопасный текст поверх ранее скопированного).

Для продвинутого сценария:
- Очистите данные расширения через страницу расширений браузера (если нужно удалить сохранённые вспомогательные значения/настройки).

---

## 6) Рекомендации для разработчика

1. **Открытый исходный код**
   - Публикуйте исходники и историю изменений.
   - Делайте воспроизводимые сборки (reproducible builds), чтобы бинарь можно было сверить с исходником.

2. **Подпись релизов**
   - Подписывайте теги/релизы (например, GPG/Sigstore).
   - Публикуйте checksums артефактов.

3. **Минимальные разрешения расширения**
   - Оставляйте только необходимые `permissions`/`host_permissions`.
   - Избегайте лишних API (`tabs`, широкие host-маски и т.п.), если они не нужны.

4. **Явное предупреждение про метаданные**
   - Шифрование защищает содержимое сообщения, но **не скрывает метаданные**:
     - кто с кем общается,
     - время сообщений,
     - частоту/размер обмена,
     - факт использования протокола.

5. **Безопасная разработка**
   - Проводите аудит криптологики и threat modeling.
   - Добавьте автоматические тесты, линтеры и проверку зависимостей в CI.
   - Не логируйте приватные ключи, raw secret, plaintext.
   - Обнуляйте/удаляйте чувствительные данные после завершения сессии.

---

## 7) Короткая памятка для пользователя (можно показать в UI)

- Отправляйте только то, что просит шаг протокола.
- Никогда не публикуйте приватные значения.
- При любой странности: остановиться → сбросить сессию → сверить собеседника вне чата → начать заново.
- Помните: содержание шифруется, но метаданные чата видимы сервису.
